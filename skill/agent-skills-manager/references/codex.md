# Codex

Load when the request names Codex, `$CODEX_HOME`, `AGENTS.md`, Codex Skills, or Codex MCP configuration.

Locate the requested workspace or user scope before editing. Skills belong to the manager workflow; `AGENTS.md`, MCP, model, and runtime configuration do not. Do not assume a current path, config filename, precedence rule, or schema: inspect locally and consult official Codex documentation if any are uncertain.

For non-Skill changes, read the active configuration, confirm the scope and target block, display a minimal diff, create a timestamped backup next to it, change only that block, validate the format or supported Codex command, and re-read the result. Preserve instructions, comments, unrelated settings, and secrets.
